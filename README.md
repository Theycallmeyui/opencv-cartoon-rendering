
# Cartoon Rendering using OpenCV

## 1. 프로젝트 개요

본 프로젝트는 OpenCV를 활용하여 입력 이미지를 만화(cartoon) 스타일로 변환하는 것을 목표로 한다.  
디지털 이미지 처리 기법을 적용하여 이미지의 윤곽선을 강조하고 색상을 단순화함으로써, 일반 이미지를 만화와 유사한 형태로 변환한다.

본 과제에서는 단순한 필터 적용이 아닌, 여러 이미지 처리 과정을 조합하여 결과를 생성하고, 해당 알고리즘의 성능 및 한계점을 분석하는 것을 목적으로 한다.

## 3. 알고리즘 설명

본 프로그램은 다음과 같은 단계로 구성된다.

Grayscale 변환 - 입력 이미지를 grayscale로 변환하여 색상 정보를 제거한다. 이는 엣지 검출을 더 효과적으로 수행하기 위함이다.

Noise 제거 (Median Blur) - Median Blur를 적용하여 이미지 내의 노이즈를 제거한다. 이 과정을 통해 불필요한 작은 변화들을 줄이고, 더 안정적인 edge detection이 가능해진다.

Edge Detection (Canny) - Canny edge detection을 이용하여 이미지의 윤곽선을 추출한다. 이 단계는 만화 스타일에서 가장 중요한 요소인 “검은 윤곽선”을 생성하는 역할을 한다.

Dilation - Dilation을 적용하여 검출된 엣지를 두껍게 만든다.
이를 통해 윤곽선이 더욱 뚜렷하게 표현되며, 만화 스타일의 특징이 강화된다.

Bilateral Filtering - Bilateral filter를 사용하여 색상을 부드럽게 만든다.
이 필터는 색상은 유지하면서도 디테일을 줄여, 만화처럼 단순화된 색 표현을 가능하게 한다.

Edge와 Color 결합 - 마지막으로, 부드럽게 처리된 색상 이미지와 엣지 이미지를 결합하여 최종 cartoon 이미지를 생성한다.

## 4. 표현 잘 안 되은 이미지 발생 (Bad Result)
   
Edge Detection 문제 
너무 많은 엣지가 검출됨,불필요한 선이 증가
원인 : 낮은 threshold 값으로 인해 작은 변화까지 엣지로 인식
결과 : 이미지가 지저분하게 보임, 윤곽선이 명확하지 않음

## 5. 해결 방법
Canny edge detection의 threshold 값을 적절히 조정하여 불필요한 엣지 검출을 줄인다.
Median Blur 또는 Gaussian Blur를 적용하여 노이즈를 제거한 후 edge detection을 수행한다.
Dilation의 kernel 크기와 iteration을 줄여 과도하게 두꺼운 선이 생성되지 않도록 조정한다.
개선 효과 - 불필요한 엣지가 감소하여 이미지가 더 깔끔해짐, 주요 윤곽선만 강조되어 만화 스타일이 더 자연스럽게 표현됨.

---

## 6. 표현이 잘 되은 이미지 발생

특징 - 배경이 단순한 이미지, 객체가 명확하게 구분되는 이미지, 조명이 충분히 밝은 이미지

결과 분석 - 이러한 조건에서는 엣지가 정확하게 검출되고, 불필요한 노이즈가 적기 때문에 만화 스타일이 자연스럽고 깔끔하게 표현된다. 특히 객체의 윤곽선이 뚜렷하게 나타나며, 색상 또한 과도하게 손실되지 않는다.
